## 📖 프로젝트 소개
### 게임 개요

Project: DawnBreaker는 플레이어가 용사가 되어 5개의 월드를 탐험하며, 각 월드의 보스를 쓰러뜨리고 황금 무기 조각을 모아 최종 보스를 격파하는 클래식 턴제 RPG입니다.

플레이어는 성장, 전직, 스킬 사용을 통해 전략적으로 전투를 진행하며, 매 스테이지마다 새로운 몬스터와 보스를 상대하게 됩니다.

### 핵심 메커니즘

⚔️ **턴제 전투 시스템**: 플레이어와 몬스터가 한 턴씩 주고받는 고전 RPG 스타일

📈 **레벨업/전직 시스템**: 검사 → 기사 → 성기사/대검사 → 성검사/용기사

🔥 **스킬/궁극기 시스템**: 직업별 고유 스킬과 궁극기를 활용한 전투

🌍 **5개 월드 × 10스테이지**: 각 월드의 보스를 쓰러뜨려 다음 지역으로 이동

---

## 🧩 게임 규칙
### 기본 플레이

플레이어는 스탯(HP/MP/공격력/마력/방어력)을 기반으로 전투합니다.

공격 또는 스킬을 선택해 몬스터를 공격합니다.

몬스터는 매 턴 플레이어에게 반격합니다.

HP가 0이 되면 게임 오버입니다.

### 전직 메커니즘

- Lv 10: 검사 → 기사

- Lv 20: 기사 → 성기사 / 대검사

- Lv 30: 성기사 → 성검사, 대검사 → 용기사

    - 전직 시 스탯 증가 및 새로운 스킬 획득

### 스킬 시스템

- 스킬은 MP를 소모합니다.

- 스킬은 물리 기반 또는 마법 기반으로 나뉩니다.

- 3차 전직 시 궁극기 해금

### 성장 시스템

- 전투 승리 → 경험치 획득

- 레벨업 시 체력·공격력이 자동 증가

- 특정 레벨에서 전직 가능

## 🌍 월드 및 스테이지 설계
| 월드  | 구성        | 특징             |
| --- | --------- | -------------- |
| 1월드 | 1~10 스테이지 | 기초 몬스터 중심      |
| 2월드 | 1~10 스테이지 | 방어력/마법 몬스터 등장  |
| 3월드 | 1~10 스테이지 | 체력 높은 몬스터 위주   |
| 4월드 | 1~10 스테이지 | 스킬 기반 강력한 중간보스 |
| 5월드 | 1~11 스테이지 | 최종보스 출현        |


- 각 월드는 1~4, 6~9 일반 몬스터,

- 5 중간 보스,

- 10 월드 보스,

- 5월드만 11스테이지 최종 보스 등장

## 🎯 오픈미션: 왜 이 주제로 했는가?
### 미션 선정 이유

>“Java 콘솔 기반에서 객체지향을 극대화한 RPG 제작”을 목표로 주제를 선정하려고 노력했습니다.

>앞선 1~3주차 프리코스에서 배운 객체지향, mvp, 의존성 주입 등등을 모두 활용할 수 있는
>
> 턴제 RPG게임은 객체지향 설계에 적합한 도메인이라고 생각했습니다.

- 전투 = 객체 간 메시지 교환
- 전직 = 상태 변화
- 스킬 = 다형성 구조
- 몬스터 = 추상화 기반 계층 구조

이를 기반으로 게임 시스템을 만들고자 했습니다.

### 개발 전략
### Phase 1 — 전투 시스템 구축 (콘솔)

- 턴 기반 전투 루프 구현

- 플레이어/몬스터 클래스 구조 완성

- 스킬/전직 시스템 로직 구축

- 콘솔 로그로 디버깅

### Phase 2 — 게임 확장 로직 구축

- 월드 → 스테이지 진행 로직

- 보스 스테이지 설계

- 플레이어 성장 모델 안정화

### Phase 3 — UI/데이터 구조 정리

- ASCII 기반 UI 디자인

- 스탯·스킬 출력 레이아웃 정리

- 전직 선택 UI 설계
